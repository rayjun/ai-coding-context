## 0 · 关于本文档
* 本文件定义 **状态文档的维护规范**。
* **实际状态文档** 位于 [docs/STATUS.md](docs/STATUS.md)。
* 目标：提供 **单一真相源**，快速恢复上下文。

---

## 1 · 核心职责
状态文档必须快速回答：

1. **现在在哪里？** → 进度 (X/Y)
2. **刚做了什么？** → 最新发现
3. **遇到什么问题？** → 阻塞列表
4. **为什么这样做？** → 决策记录
5. **下次做什么？** → 下一步指南

---

## 2 · 必须包含的章节
### 2.1 元信息
```markdown
> **最后更新**: YYYY-MM-DD HH:MM UTC
> **当前阶段**: [阶段名称]
> **整体进度**: X/Y 任务完成 (Z%)
```

### 2.2 当前目标
```markdown
## 当前目标
[1-2 句话]
**参考**: [规划文档链接]
```

### 2.3 任务进度
```markdown
## 任务进度 (X/Y)

### 已完成
#### Task-X: [名称]
**时间**: YYYY-MM-DD
**结果**: [3-5 条要点]
**报告**: [链接]

### 进行中
#### Task-Y: [名称]
**状态**: [进展]
**下一步**: [行动]

### 待办
#### Task-Z: [名称]
**目标**: [说明]
**前置**: [依赖]
```

### 2.4 最新发现
```markdown
## 最新发现
- [发现 1 + 数据]
- [发现 2 + 数据]
- 详见: [报告链接]
```

### 2.5 阻塞问题
```markdown
## 阻塞问题

### 无阻塞

或：

### 问题 #N: [简述]
**影响**: [任务]
**根因**: [原因]
**方案**: [解决方案]
**状态**: [紧急/中等/已解决]
```

### 2.6 关键决策
```markdown
## 决策记录

### 决策 #N: [简述]
**日期**: YYYY-MM-DD
**背景**: [为什么]
**决策**: [是什么]
**理由**: [原因列表]
**影响**: [后果]
```

### 2.7 下次会话指南
```markdown
## 下次从这里开始

### 恢复上下文
1. 读取 `docs/STATUS.md`
2. 查看 [最新报告]

### 继续工作
"[具体行动]"

### 推荐选项
**A (推荐)**: [方案] - 优点/缺点
**B**: [方案] - 优点/缺点
**C**: [方案] - 优点/缺点
```

## 3 · 更新时机
| 事件 | 更新内容 |
|------|---------|
| 完成任务 | 进度、已完成、发现、下一步 |
| 代码修改 | 最近修改 |
| 做决策 | 决策记录 |
| 遇阻塞 | 阻塞问题 |
| 目标变更 | 当前目标 |
| 会话结束 | 时间戳、下一步 |

---

## 4 · 书写原则
### DO
* 简洁直接："Task-X 完成"
* 用数据："82/82 通过 (100%)"
* 明确状态："2/6 完成 (33%)"
* 链接报告，不复制内容
* 每次会话更新

### DON'T
* 不重复其他文档内容
* 不包含实现细节
* 不用模糊语言："差不多"、"快了"
* 不忘更新时间戳
* 不删除历史决策
